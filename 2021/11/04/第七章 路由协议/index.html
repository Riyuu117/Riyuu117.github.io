第七章 路由协议

发表于2021-11-04|更新于2021-11-04

静态路由与动态路由

静态路由：事先设置好路由器和主机并将路由信息固定的一种方法。

动态路由：让路由协议在运行的过程中自动地设置路由控制信息的一种方法。

静态路由和动态路由比较：

路由控制范围

自治系统（AS，Autonomous System）或路由选择域（Routing Domain）：制定自己的路由策略，并以此为准在一个或多个网络群体中采用的小型单位，即对路由控制范围的分层。

根据路由的控制范围，AS内部动态路由采用内部网关协议（域内路由协议）IGP，AS之间的路由控制采用外部网关协议（域间路由协议）EGP。IGP和EGP的关系可以类比网络地址和主机地址。

RIP

概述

  • RIP（Routing Information Protocol）路由选择协议；
  • 属于域内路由器协议；
  • 基于距离向量算法；
  • 特点：简单，但收敛慢；
  • 适用场景：小型网络（LAN）；
  • RIP协议要求网络中每一个路由器都维护从它自己到其他每个目的网络的唯一最佳距离记录（即一组距离）；
  • 距离（Metrics）：通常为“跳数”，即从源端口到目的端口所经过的路由器个数，经过一个路由器+1。特别的，从一路由器到直接连接的网络距离为1。RIP允许一条路由最多只包含15个路由器，因此距离为16表示网络不可达！

路由信息交换

仅和相邻路由器交换信息；

路由器交换的信息是自己的路由表；

每30秒交换一次路由信息，然后路由器根据新信息更新路由表。若超过180秒没收到邻居路由器的通告，则判定邻居没了，并更新自己路由表；

收敛：路由器刚开始工作时，只知道直接连接的网络距离为1，接着每一个路由器也只和数量非常有限的相邻路由器交换并更新路由信息。经过若干次更新后，所有路由器最终都会达到本自治系统任何一个网络的最短距离和下一跳路由器的地址。

区分：路由器通过广播在本自治系统内发送自己的路由表。但和同一链路广播不同，该广播只发送给相邻路由器，在相邻路由器更新后再将自己的路由表接力发送，体现为仅和相邻路由器交换信息！

距离向量算法

路由表的组成：

  • 显示该路由器到本自治系统内任何网段的路径；
  • 如果到自己直连的网段，则下一个路由为本路由器对应的接口；
  • 如果到非直连的其他网段，则下一个路由为相邻路由器对应的接口；

距离向量算法过程：

  1. 接收到相邻路由器发来的路由表；

  2. 修改该路由表的所有表项：对地址为X的相邻路由器发来的路由表，把“下一跳”字段中的地址改为X，并把所有的“距离”字段+1；

  3. 参考自己的路由表和修改后的路由表：

    R1路由表中若没用Net3，则把项目填入R1路由表

    R1路由表中若有Net3，则查看下一跳路由器地址：

    ​ 若下一跳是X，则用收到的项目替换源路由表中的项目；

    ​ 若下一跳不是X，原来距离比从X走的距离远则更新，否则不处理；

  4. 若180s还没有收到相邻路由器X的更新路由表，则把X即为不可达路由器，即把距离设置为16。

问题及解决

收敛慢：当网络出现故障时，要经过较长的时间（例如数分钟）才能将此消息传送到所有路由器。

收敛慢的本质：路由器会把收到的路由消息原路返回给发送端。

例如下图：路由器A告诉路由器B，它到网络A的距离为1。路由器B接受到消息后更新路由表，经过一定时间后，路由器B又告诉路由器A，它到网络A的距离为2。

没有环路

解决：水平分割，路由器不会把收到的路由消息原路返回给发送端。

有环路

在环路中，反向的回路会构成迂回的通道，光靠水平分割不能解决问题。

路由器B：是路由器C、D告诉我，到网络A的距离为3！ 因此仍可以将未更新的报文发送给路由器A！

解决：毒性逆转：通信故障时，路由器发送一个距离为16的消息；

​ 触发更新：路由表信息发送变化时，不等待30秒，马上发送，加快收敛。

OSPF

概述

  • OSPF（Open Shortest Path First）开放式最短路径优先协议；

  • 属于域内路由器协议；

  • 基于链路状态路由算法；

  • 特点：收敛快，但网络规模逐渐越大时，处理时间越长，对CPU和内存的消耗也越大。

  • 适用场景：大型网络；

  • OSPF的特点：

    每个路由器会将自己的链路状态向相邻的所有路由器广播，最终每个路由器都有相同的网络拓扑信息，然后根据最短路径优先算法（Dijkstra）生成路由表！

链路状态路由算法

链路状态路由算法是在了解网络整体连接状态的基础上生成路由控制表的一种方法，每个路由器必须保存同样的信息才能进行正确的路由选择。

OSPF的区域

  • 为使OSPF能够用于规模很大的网络，OSPF将一个自治系统再划分为若干更小的范围，称为区域。
  • 划分区域的好处是，将利用洪泛法交换链路状态信息的范围局限于每个区域而非整个自治系统，减少了整个网络上的通信量。
  • 在一个区域内部的路由器只知道本区域的完整网络拓扑，关于区域之外的路径信息，只能从区域边界路由器获取，且只获取区域边界路由器达到这些路由器的距离信息。
  • 这些区域也有层次之分。处在上层的域称为主干区域，负责连通其他下层的区域，并且还连接其他自治域。

BGP

  • BGP（Border Gateway Protocol）边界网关协议；

  • 属于域间路由器协议；

  • 基于路径向量算法；

  • 特点：刚刚运行时，BGP的邻站是交换整个BGP路由表，但以后只需要在发送变化时更新有变化的部分。

  • 信息交换细节：

    ​ 和谁交换？与其他AS的邻站BGP扬声器交换信息；

    ​ 交换什么？交换的网络可达性的信息，即要到达某个网络所要经过的一些列AS。当到达目标网络时，会生成一个中途经过所有AS的编号列表（AS路径信息访问列表），基于AS之间的合约选择合适的路由；

    ​ 交换基准？AS之间的合约（是否允许中转、中转费用）；

    ​ 多久交换？发生变化时更新有变化的部分；

    ​ 下层协议？在不同自治系统之间交换路由信息，由于网络环境复杂，需要保证可靠传输，因此采用TCP；

RIP、OSPF和BGP的区别

关于RIP和OSPF比较的补充：

  • 交换的细节：RIP中包的类型只有一种，当网络规模越大，每次要交换的路由控制信息就越大，而且当网络已经处于比较稳定、没用什么变化的状态时，还是要定期交换相同的路由控制信息，浪费了网络带宽；而OSPF交换的链路状态包类型有5种，包长度小，且仅在链路状态发送改变时才交换信息，因此大大地减少网络流量，达到迅速更新路由信息的目的。

  • 路径选择：RIP选择路由器个数较少的路径；OSPF选择总的代价较小的路径。
