第五章 IP相关的技术

发表于2021-10-30|更新于2021-10-31
|字数总计:2k|阅读时长:6分钟|阅读量:

DNS

DNS域名系统（Domain Name System）

  • 将字符串自动转换为具体的IP地址，有效管理域名和IP地址之间的对应关系；
  • 解决IP地址不便记忆的问题。

域名服务器和解析器

域名服务器：

  • 管理域名的主机和相应的软件，可以管理所在分层的域的相关信息；
  • 每个域的分层上都有自己的域名；
  • 各层域名服务器都了解该层以下的分层中所有域名服务器的IP地址；
  • 所有域名服务器都了解域名服务器的IP地址；

解析器：

  • 进行DNS查询的主机和软件。

DNS查询

  • 解析器不仅可以访问同一域中的域名服务器，还可以访问其他域的域名服务器；
  • 解析器和域名服务器会将最新了解到的信息临时保存在缓存中，减少每次查询的性能消耗。

ARP

概要

  • ARP（Address Resolution Protocol）属于数据链路层协议；
  • 以目标IP地址为线索，用来定位下一个应该接收数据分包的网络设备对应的MAC地址；
  • 如果目标主机不在同一个链路上时，可以通过ARP查找下一跳路由器的MAC地址；
  • ARP只适用于IPv4，不能用于IPv6。IPv6中可以可以用ICMPv6替代ARP发送邻居探索消息。

工作机制

  • 借助ARP请求和ARP响应两种类型确定MAC地址；
  • 源主机将自己的MAC地址和目的主机的IP地址塞进ARP请求，通过广播发送一个ARP请求包，该包可以被同一链路上所有的主机或路由器接收；
  • 目的主机将自己的MAC地址填入其中的ARP响应包返回IP地址；
  • 主机或路由器可以将获取到的MAC地址缓存一段时间到ARP缓存表中，可以节约网络流量。

RARP

从MAC地址定位IP地址，作用是完成硬件地址到IP地址的映射；

主要用于一些没有任何输入接口或无法通过DHCP动态获取IP地址的小型嵌入式设备；

工作流程：在网络中配置一台RARP服务器，里面保存着IP地址和MAC地址的映射关系，当无盘工作站启动后，就封装一个RARP数据包，里面有其MAC地址，然后广播到网络上去，当服务器收到请求包后，就查找对应的MAC地址的IP地址装入响应报文中发回给请求者；

因为需要广播请求报文，因此RARP只能用于具有广播能力的网络。

ICMP

用于辅助IP的ICMP

  • ICMP（Internet Control Message Protocol）；

  • 主要功能：确定IP包是否成功送达目标地址，通知在发送过程中IP包被废弃的具体原因，改善网络设置等；

  • 在IP通信中如果某个IP包因为某种原因未能到达目标地址，那么这个具体原因将由ICMP负责通知；

  • ICMP的这种通知消息会使用IP进行发送，在ICMP中，包以明文的形式像TCP/UDP一样通过IP进行传输。然而，ICMP所承担的功能并非传输层的补充，而应该把它考虑为IP的一部分。

主要的ICMP消息

ICMP报文分类

  • 差错报文：主要用于回应网络错误信息。比如目标不可达（类型值为3）和重定向（类型值为5）；
  • 查询报文：主要用来查询网络信息。比如ping程序使用ICMP报文来查看目标是否可达（类型值为8）。

ICMP消息类型

ICMP报文使用8位代码字段进步一细分不同的条件，使用16位校验和字段（包括头部和内容部分）进行循环冗余校验，以检验报文在传输过程中是否被损坏。

ICMP超时消息（类型11）：

  • IP包中有一个TTL（Time To Live 生存周期 ）的字段，其值随着每经过一个路由器会减1，直到减到0时该IP包会被丢弃，此时路由器发送一个ICMP超时消息给源主机，通知该包已经被丢弃；
  • 避免路由器控制发送循环状况导致IP包被无休止转发 和 控制包到达范围。

ICMPv6

  • 在IPv6中，从IP地址定位MAC地址的协议从ARP转为ICMP的邻居探索消息（Neighbor Discovery）；
  • 分成两大类：错误消息和信息消息，0-127属于错误消息，128-255属于信息消息。
  • 主要功能：邻居探索和自动获取IP。

    ICMP仅作为IPv4的辅助技术，没有仍可以通信。但没有ICMPv6，IPv6就无法进行正常通信！

邻居探索

  • 邻居请求消息：用于查询IPv6地址与MAC地址对用关系，并由邻居宣告消息得知MAC地址；

  • 邻居探索消息：ICMPv6从类型133到类型137的消息；

IP的自动配置

  • 实现即插即用作用，在没有DHCP服务器的环境下也能实现IP地址的自动获取；

  • 没有路由器：使用MAC地址作为链路本地单播地址；

  • 有路由器：从路由器中获得IPv6地址前面部分，后面用MAC地址配置。

DHCP

概要

  • DCHP（Dynamic host configuration protocol）；

  • 功能：自动设置IP地址，统一管理IP地址分配；

  • 管理员只要在DCHP服务器上做一些必要的设定，DCHP服务器就会保证IP地址的唯一性，减轻了管理员的负担。

工作机制

  1. 架设两台或两台以上的DCHP服务器；

  2. 将DCHP所要分配的IP地址范围设置到服务器上；

  3. 再将相应的子网掩码、路由控制信息、DNS服务器地址等设置到服务器上。

    • DHCP服务器：在分配IP地址前发送ICMP会送请求包，确认没有返回应答；

    • DHCP客户端：针对从DHCP那里获得IP地址发送ARP请求包，确认没有返回应答。

DHCP分配IP地址的方法

  1. 由DHCP服务器在特定的IP地址中自动选出一个进行分配
  2. 针对MAC地址分配一个固定的IP地址。

DHCP中继代理

  • 只有一个以太网（无线LAN）的网段，通常用宽带路由器作为DHCP服务器；

  • 多个以太网（无线LAN）网段，通常架设一个DHCP服务器，使不在同一链路上也可以统一分配和管理IP地址，同一链路由DHCP中继代理实现；

  • DHCP客户端——>DHCP中继代理——>DHCP服务器。

NAT

概要

NAT（Network Address Translator）：在本地网络中使用私有地址，在连接互联网时转而使用全局IP地址；

NAPT（Network Address Port Translator）：转换IP地址和转换TCP、UDP端口号，实现一个全局IP地址和多个主机的通信。

通常提到的NAT指的是NAPT！

工作机制

NAT：

NAPT：将不同主机的IP地址不同—> 端口号不同

同一通信连接满足的条件：源IP地址、目的IP地址、源端口号、目的端口号、协议类型

潜在的问题

  • 无法从NAT的外部向内部服务器建立连接；
  • 转换表的生成与转化操作都会产生一定开销；
  • 通信过程中一旦NAT遇到异常需要重新启动时，所有TCP连接都将被重置；
  • 即使备置两台NAT做容灾备份，TCP还是会被断开。
